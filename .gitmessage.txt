Type: 한국말로 적든 영어로 적든 50자 이내의 summary를 쓰자.

Type으로 사용할 수 있는 것들은 다음과 같다.
Feat: 새로운 기능
New : 완전 새로운 기능
Improve : 기존 기능에서 수정, 요구사항 변경으로인한 수정
Fix: 버그 수정
Docs: 문서 변경
Style: 코드 포맷팅, 세미콜론 추가 등(코드 변경 없음)
Refactor: 프로덕션 코드 리팩토링
Test: 테스트, 리팩토링 작업, 프로덕션 코드 변경 없음
Chore: 빌드작업 업데이트, 패키지 관리자 구성 변경 등 프로덕션 코드 변경 없음
Release : 릴리즈 배포 버전 기입용

본문은 1줄의 blank line 이후에 작성한다.

- Bullet point, hyphen, asterisk를 입력하고 한 칸을 띄어쓰기하면 •와 같이 출력된다.

- Hanging indent를 사용하자
